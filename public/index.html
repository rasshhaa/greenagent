Core Intelligence

Every feature built around future impact

Two levels: MR-level diff analysis and whole-repo push analysis with a real dependency graph built from your actual import statements.

Real Dependency Graph

Built from actual import/require statements via static analysis. Nodes = your files. Edges = real imports. Coloured by layer (routes, services, data, utils) and status (hotspot, hub, stable).

Cross-File Energy Analysis

Catches N+1 patterns, missing caching layers, and heavy sync ops that span multiple modules — the kind of inefficiency invisible to single-file linters.

Hotspot File Detection

Files ranked by in-degree (many dependents — change risk) and out-degree (many imports — coupling risk). LLM validates and explains each hotspot.

Architectural Layer Analysis

Files are automatically classified into layers (routes, services, models, data, middleware, utils, config) from their paths — giving the LLM a structural map of your system.

The Process

Push → graph → score → report

01
Push to GitLab

Developer pushes commits. GitLab fires a Push Hook to GreenAgent.

02
Fetch entire repo

GreenAgent fetches every source file via GitLab API — up to 120 files, priority-sorted by architectural importance.

03
Build real graph

Static import analysis extracts all import/require statements and resolves them to actual file paths — building a real dependency graph.

04
Score + report

Groq (Llama 3.3 70B) analyses the whole codebase, identifies hotspots, and posts a sustainability report on the commit.

Configuration

Setup & Connect

Get GreenAgent connected in under 5 minutes. Both MR Hook and Push Hook register automatically.

01
Install & configure

Fill in your keys and start the server.

cp .env.example .env
npm install
npm start
GROQ_API_KEYconsole.groq.com → API Keys
GITLAB_OAUTH_CLIENT_IDGitLab → Applications → New (set redirect to TUNNEL_URL/auth/gitlab/callback)
GITLAB_OAUTH_CLIENT_SECRETGenerated when you create the GitLab OAuth app
GITLAB_OAUTH_REDIRECT_URIe.g. https://your-ngrok-url.ngrok-free.dev/auth/gitlab/callback
GITLAB_WEBHOOK_SECRETAny random string
02
Expose with ngrok

GitLab needs a public URL to send webhooks.

npx ngrok http 3000
03
Login with GitLab

Click "Get Started" → "Continue with GitLab". GreenAgent registers webhooks on all your repos automatically.

MR Hook → /webhook/gitlab
Push Hook → /webhook/gitlab

Verify
